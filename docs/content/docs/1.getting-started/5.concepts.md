## MapboxContext 注入

`MapboxMap` 创建地图后，通过 Vue 的 `provide` 下发一个 **MapboxContext**。子组件（`MapboxSource`、`MapboxLayer`、控件等）经 `useMap()` 直接拿到上下文，无需手动传递 id 或查表。

```ts
const { map, isLoaded, whenLoaded, onReady } = useMap()
```

| 字段 | 说明 |
| --- | --- |
| `map` | 地图实例的 `ShallowRef`，SSR 与挂载前为 `undefined` |
| `isLoaded` | 样式是否已加载完成 |
| `whenLoaded()` | 样式首次加载完成时 resolve，返回 `Promise<Map>` |
| `onReady(cb)` | 样式就绪即执行，且每次 `setStyle` 重载后重跑（用于重建 source/layer），返回取消订阅函数 |

::warning
`useMap()` 必须在 `<MapboxMap>` 子树内调用，否则抛错。需要跨树或跨路由访问时改用 `useMapbox(id)`。
::

### useMapbox —— 跨树逃生口

为 `MapboxMap` 指定 `map-id` 后，可在任意位置经 `useMapbox(id)` 从注册表获取同一上下文：

```vue
<MapboxMap map-id="main" :options="{ style }" />
```

```ts
const { whenLoaded } = useMapbox('main')
const map = await whenLoaded()
```

## 相机双向绑定

`MapboxMap` 用 `defineModel` 暴露四个相机模型，`moveend` 后回写，支持 `v-model` 双向绑定：

```vue
<script setup lang="ts">
const center = ref<[number, number]>([116.397, 39.908])
const zoom = ref(9)
const bearing = ref(0)
const pitch = ref(0)
</script>

<template>
  <MapboxMap
    v-model:center="center"
    v-model:zoom="zoom"
    v-model:bearing="bearing"
    v-model:pitch="pitch"
    :options="{ style: 'mapbox://styles/mapbox/streets-v12' }"
  />
</template>
```

外部修改 `center` / `zoom` 等会驱动地图移动，地图交互又会同步回这些 ref。

## SSR 安全

地图实例只在客户端 `onMounted` 创建。`MapboxSource` / `MapboxLayer` 等子组件在 SSR 与挂载前不会触碰地图实例，因此**在 Nuxt 下无需 `<ClientOnly>` 包裹**。如需跳过服务端占位渲染，可自行选用。

## 跨路由持久化

`MapboxMap` 设置 `persistent` 后，组件卸载时不销毁地图实例，配合路由 keepalive / `<KeepAlive>` 可跨路由复用同一张地图，避免重复初始化开销：

```vue
<MapboxMap persistent map-id="shared" :options="{ style }" />
```
