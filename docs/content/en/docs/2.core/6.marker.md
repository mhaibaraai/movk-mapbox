## Introduction

`MapboxMarker` places a marker on the map: the default slot provides custom DOM content (omitting it falls back to the mapbox default teardrop icon), and the position is two-way bound via `v-model:lnglat`. When `options.draggable` is enabled, the new coordinates are written back to the bound value after each drag ends.

Provide a `#popup` slot to attach a popup rendering any Vue component. `trigger` controls how it opens (`click` / `hover` / `none`), and the open state is exposed through `v-model:open`.

::note
With `trigger="click"`, the click on the marker does not bubble to the map, so `MapboxMap` emits no `click` event for it. This is required: marker elements live inside the map's canvas container, and letting the click bubble would fire mapbox's `preclick`, closing the just-opened popup via `closeOnClick`.
::

::note
With `trigger="hover"`, moving the cursor into the popup fires the marker's `mouseleave` and closes it â€” an inherent limitation of the popup and marker being separate DOM nodes. For hover, prefer a small `popupOptions.offset`; if the popup needs interaction, use `click`.
::
