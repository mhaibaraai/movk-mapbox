## Introduction

`heatmapPaint` generates a complete heatmap layer paint in one call: it linearly maps a specified feature property to a 0..1 heat weight via `weightRange`, and colors the result by `heatmap-density` through color stops (density 0 is automatically transparent to avoid flooding the entire map with a background color). This eliminates the need to hand-write verbose `interpolate` expressions.

Import from `@movk/mapbox/utils/heatmap` (not auto-imported).

## API

### `heatmapPaint()`{lang="ts-type"}

Generate a heatmap layer paint object.

::field-group
  ::field{name="options.weightProperty" type="string"}
  Feature property used as the weight value, default `'temperature'`.
  ::
  ::field{name="options.weightRange" type="[number, number]"}
  Value range `[min, max]` of the weight property, linearly mapped to 0..1, default `[0, 40]`.
  ::
  ::field{name="options.colorStops" type="[number, string][]"}
  Density-color stops `[density 0..1, color][]`; defaults to a five-step blue→red ramp with density 0 automatically transparent.
  ::
  ::field{name="options.radius" type="number"}
  Heatmap radius in pixels, default `30`.
  ::
  ::field{name="options.intensity" type="number"}
  Heatmap intensity, default `1`.
  ::
  ::field{name="options.opacity" type="number"}
  Overall opacity, default `1`.
  ::
::

Returns a heatmap layer paint object that can be bound directly to the `paint` prop of `<MapboxLayer type="heatmap">`.
