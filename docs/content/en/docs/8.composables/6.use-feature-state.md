## Introduction

`useFeatureState` automatically maintains `feature-state` hover and selected states on a target layer. The consumer-side paint uses `['feature-state', 'hover' | 'selected']` expressions to drive styling, enabling hover highlighting and selection without resetting the source data. Returns the currently hovered and selected features along with a method to clear the selection.

::note{color="warning"}
Features must have an `id`: configure `generateId: true` or `promoteId` on the source, otherwise `feature-state` cannot locate the feature.
::

## API

### `useFeatureState()`{lang="ts-type"}

Maintains interaction state on a target layer.

::field-group
  ::field{name="layerId" type="string" required}
  The target layer id.
  ::
  ::field{name="options.mapId" type="string"}
  Target map id; required when used outside a `<MapboxMap>` subtree.
  ::
  ::field{name="options.hover" type="boolean"}
  Maintain hover state, defaults to `true`.
  ::
  ::field{name="options.select" type="boolean"}
  Maintain click-selection state, defaults to `true`.
  ::
  ::field{name="options.cursor" type="boolean"}
  Set a pointer cursor on hover, defaults to `true`.
  ::
::

Returns `UseFeatureStateReturn`:

::field-group
  ::field{name="hovered" type="ShallowRef<GeoJSONFeature | undefined>"}
  The currently hovered feature.
  ::
  ::field{name="selected" type="ShallowRef<GeoJSONFeature | undefined>"}
  The currently selected feature.
  ::
  ::field{name="clearSelection" type="() => void"}
  Clears the current selection.
  ::
::
