## Introduction

`useMap` injects the context provided by the nearest `<MapboxMap>`, returning a reference to the map instance, the style-loaded state, and a ready callback. It is the unified entry point for all declarative child components to access the map, without requiring an id lookup.

::note
Must be called inside a `<MapboxMap>` subtree, otherwise it will throw. To access the map outside the component tree, use [useMapbox](/docs/composables/use-mapbox) instead.
::

`onReady` re-runs on every `setStyle` call as well as the initial style load, making it the recommended place to create sources, layers, and bind events. `map` is `undefined` during SSR and before mount — the component handles SSR safety internally.

## API

### `useMap()`{lang="ts-type"}

Injects the current map context and returns `MapboxContext`. No parameters.

Returns `MapboxContext`:

::field-group
  ::field{name="id" type="string"}
  The map id; auto-generated when not explicitly set.
  ::
  ::field{name="map" type="ShallowRef<Map | undefined>"}
  Reference to the map instance; `undefined` during SSR and before mount.
  ::
  ::field{name="isLoaded" type="Ref<boolean>"}
  Whether the map style has finished loading.
  ::
  ::field{name="whenLoaded" type="() => Promise<Map>"}
  Resolves when the style has loaded for the first time, returning the map instance.
  ::
  ::field{name="onReady" type="(cb: (map: Map) => void) => () => void"}
  Executes the callback when the style is ready, and re-runs it after each `setStyle` reload (for rebuilding sources and layers). Returns an unsubscribe function.
  ::
::
